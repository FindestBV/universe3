# Introduction to Tailwind CSS

Tailwind CSS is a utility-first CSS framework that provides low-level utility classes to build modern, responsive designs directly in your HTML. Unlike traditional frameworks like Bootstrap, which offer predefined components, Tailwind gives you complete control to style your elements without leaving your markup.

This guide introduces you to Tailwind CSS, how to configure `tailwind.config.js`, and how to create custom classes using Tailwind's directives.

### Reference:

[Tailwind Docs](https://tailwindcss.com/docs/installation)

## Why Use Tailwind CSS?

### Key Features
1. **Utility-First Design**:
   - Tailwind offers hundreds of utility classes (e.g., `bg-red-500`, `text-xl`, `flex`, etc.) that you can use to design directly in your markup.
2. **Customization**:
   - Tailwind's configuration allows you to customize colors, spacing, fonts, and more.
3. **Responsive Design**:
   - Built-in responsive design utilities (`sm:`, `md:`, `lg:`, `xl:`) make it easy to adapt your site for any screen size.
4. **Performance**:
   - Tailwind includes a built-in `purge` feature to remove unused styles, reducing your CSS file size.

## Setting Up Tailwind CSS

### Installation

1. **Install Tailwind via npm**:
   ```bash
   npm install -D tailwindcss postcss autoprefixer
   ```

2. **Initialize Tailwind**:
   Generate a `tailwind.config.js` file:
   ```bash
   npx tailwindcss init
   ```

3. **Setup `postcss.config.js`**:
   Add the following:
   ```javascript
   module.exports = {
     plugins: {
       tailwindcss: {},
       autoprefixer: {},
     },
   };
   ```

4. **Include Tailwind in Your CSS**:
   Create a CSS file (e.g., `src/styles.css`) and add the Tailwind directives:
   ```css
   @tailwind base;
   @tailwind components;
   @tailwind utilities;
   ```

5. **Update Build Config**:
   Ensure your build tool (e.g., Vite, Webpack) processes the Tailwind CSS file.

## Configuring `tailwind.config.js`

The `tailwind.config.js` file is the heart of Tailwind's customization. Here's an example configuration:

```javascript
module.exports = {
  content: [
    './src/**/*.{html,js,ts,jsx,tsx}', // Define where Tailwind should look for classes
  ],
  theme: {
    extend: {
      colors: {
        primary: '#4A90E2',
        secondary: '#D0021B',
      },
      spacing: {
        '72': '18rem',
        '84': '21rem',
        '96': '24rem',
      },
      fontFamily: {
        sans: ['Inter', 'sans-serif'],
      },
    },
  },
  plugins: [],
};
```

### Key Sections

1. **`content`**:
   - Specifies where Tailwind scans for class names to include in the final CSS.

2. **`theme`**:
   - Allows you to extend or override the default Tailwind theme.

3. **`extend`**:
   - Add custom colors, spacing, fonts, and more without replacing the entire theme.

4. **Plugins**:
   - Use or create plugins for additional functionality.

## Customizing with Tailwind Directives

### Creating Custom Utility Classes

You can create reusable styles using Tailwind's directives in your CSS file.

1. **Using `@apply`**:
   - Combine multiple utility classes into a custom class.
   ```css
   .btn {
     @apply bg-blue-500 text-white font-bold py-2 px-4 rounded;
   }
   .btn-primary {
     @apply btn bg-blue-700 hover:bg-blue-600;
   }
   .btn-secondary {
     @apply btn bg-gray-700 hover:bg-gray-600;
   }
   ```

2. **Adding Custom Styles**:
   - Extend the default theme to add new utilities.
   ```javascript
   module.exports = {
     theme: {
       extend: {
         boxShadow: {
           'outline-blue': '0 0 0 3px rgba(59, 130, 246, 0.5)',
         },
       },
     },
   };
   ```

   Then use it in your HTML:
   ```html
   <div class="shadow-outline-blue"></div>
   ```

## Responsive Design with Tailwind

Tailwind's responsive utilities make it easy to adapt designs for different screen sizes. Prefix a utility class with the screen size modifier:
- **Breakpoints**: `sm`, `md`, `lg`, `xl`, `2xl`

```html
<div class="text-sm sm:text-base md:text-lg lg:text-xl">
  Responsive text
</div>
```

## Purging Unused CSS

To remove unused styles and optimize performance, configure the `purge` option in `tailwind.config.js`:
```javascript
module.exports = {
  content: ['./src/**/*.{html,js,jsx,ts,tsx}'], // Files to scan for classes
  theme: {
    extend: {},
  },
  plugins: [],
};
```

Tailwind will remove unused classes in production builds.
